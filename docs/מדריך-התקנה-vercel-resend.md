# מדריך התקנה מלא — Vercel + Resend

מדריך קבוע לשימוש חוזר עם כל לקוח.
מסביר צעד-אחר-צעד איך מעלים אתר Next.js לאוויר ב-**Vercel** ומחברים שליחת מייל דרך **Resend**.
כתוב בשפה פשוטה ככל האפשר; חלקים טכניים מסומנים ב-🔧.

> מה צריך לפני שמתחילים:
> - הקוד של האתר במאגר Git (GitHub / GitLab / Bitbucket).
> - חשבון מייל פעיל (אליו יגיעו הפניות).
> - 20–30 דקות.

---

# חלק א' — Vercel (אחסון והעלאה לאוויר)

## מה זה Vercel ולמה דווקא הוא
Vercel הוא השירות שמאחסן את האתר ומגיש אותו לגולשים. הוא נבנה ע"י יוצרי Next.js,
ולכן אתר Next.js עובד עליו בלי הגדרות. היתרונות:
- אין שרת פיזי לתחזק, אין עדכוני אבטחה ידניים.
- נטען מהר מכל העולם (רשת CDN גלובלית).
- כל דחיפה (push) לקוד עולה לאוויר אוטומטית.
- **חינם** לאתרי תדמית.

---

## שלב 1 — להעלות את הקוד ל-GitHub
אם הקוד עוד לא ב-GitHub:
1. נכנסים ל-**github.com**, נרשמים/מתחברים.
2. יוצרים מאגר חדש (**New repository**) — שם באנגלית, למשל `tzahi-site`.
3. 🔧 מהמחשב, בתיקיית הפרויקט:
   ```bash
   git init
   git add .
   git commit -m "initial"
   git branch -M main
   git remote add origin https://github.com/USER/tzahi-site.git
   git push -u origin main
   ```

---

## שלב 2 — לפתוח חשבון Vercel
1. נכנסים ל-**vercel.com** → **Sign Up**.
2. נרשמים עם **GitHub** (מומלץ — מחבר אוטומטית את המאגרים).
3. בוחרים בתוכנית **Hobby** (חינם) להתחלה.

> הערה: התוכנית החינמית (Hobby) מצוינת לאתר תדמית. אם בעתיד צריך לצרף כמה
> אנשי צוות לניהול — יש תוכנית Pro (בערך ‎$20 לחודש). לא נדרש להתחלה.

---

## שלב 3 — לייבא את הפרויקט (Import)
1. בלוח של Vercel: **Add New… → Project**.
2. ברשימת המאגרים בוחרים את `tzahi-site` ולוחצים **Import**.
3. Vercel מזהה אוטומטית שזה **Next.js** וממלא לבד:
   - Framework: Next.js
   - Build Command: `next build`
   - Install Command: `npm install`
   - אין מה לשנות.
4. **אל תלחץ Deploy עדיין** — קודם מוסיפים את משתני הסביבה (שלב 4).

---

## שלב 4 — משתני סביבה (Environment Variables) 🔧
אלה "הגדרות סודיות" שהאתר משתמש בהן (כמו מפתח המייל). לעולם לא נכתבות בתוך הקוד.

במסך ה-Import (או אחר כך ב-**Settings → Environment Variables**) מוסיפים:

| שם המשתנה | ערך | חובה? |
|---|---|---|
| `RESEND_API_KEY` | המפתח מ-Resend (מתחיל ב-`re_`) — נוציא אותו בחלק ב' | ✅ חובה למייל |
| `RESEND_TO` | המייל לקבלת פניות (אם ריק — ברירת מחדל מהקוד) | אופציונלי |
| `RESEND_FROM` | כתובת השולח (אם ריק — `onboarding@resend.dev`) | אופציונלי |

- אם עדיין אין מפתח Resend — אפשר לפרוס עכשיו בלי `RESEND_API_KEY` ולהוסיף בהמשך
  (הטופס פשוט יפנה את הגולש לוואטסאפ/טלפון עד שמוסיפים מפתח).
- כל משתנה אפשר להחיל על Production / Preview / Development. לאתר תדמית — סמן את שלושתם.

---

## שלב 5 — Deploy
1. לוחצים **Deploy**.
2. ממתינים 1–2 דקות (Vercel בונה ומעלה).
3. מקבלים כתובת זמנית חינמית, למשל `tzahi-site.vercel.app` — האתר באוויר! 🎉

> מאותו רגע: **כל push ל-main מעלה גרסה חדשה אוטומטית.** אין צורך להעלות ידנית.

---

## שלב 6 — חיבור דומיין (כתובת מקצועית)
כדי שהאתר יהיה על כתובת אמיתית (למשל `tzahi-tzedaka.co.il`) ולא על `*.vercel.app`:

**Settings → Domains → Add Domain**, ואז אחת משתי דרכים:

### דרך א' — לקנות דרך Vercel (הכי פשוט) ⭐
- מקלידים את השם, רואים אם פנוי, קונים בלחיצה. Vercel מחבר הכל אוטומטית.
- עלות: בערך ‎$15–20 לשנה.

### דרך ב' — דומיין שכבר קיים אצל ספק חיצוני 🔧
מוסיפים את הדומיין ב-Vercel ואז מגדירים אצל ספק הדומיין את ה-DNS:
- לכתובת הראשית (apex, למשל `tzahi-tzedaka.co.il`): רשומת **A** →‏ `76.76.21.21`
- לתת-דומיין `www`: רשומת **CNAME** →‏ `cname.vercel-dns.com`
- (לחלופין: מצביעים את ה-**Nameservers** של הדומיין ל-Vercel ונותנים לו לנהל הכל)
- ההפעלה לוקחת בין כמה דקות לכמה שעות (עד שה-DNS מתעדכן). Vercel מנפיק תעודת SSL (https) אוטומטית.

---

## שלב 7 — בעלות ושיתוף (Collaboration)
כדי שהבעלות תהיה של הלקוח ולא תלויה במפתח:

- **הזמנת הלקוח לצוות:** Settings → **Members** → Invite (דורש תוכנית Team/Pro למספר חברים).
- **העברת בעלות מלאה:** Project → Settings → **Advanced → Transfer Project** →
  לחשבון או לצוות של הלקוח. הלקוח פותח חשבון Vercel חינמי, ומעבירים אליו.

> ההמלצה: הלקוח פותח חשבון Vercel משלו, והאתר מועבר אליו. כך הוא הבעלים לתמיד,
> גם אם יחליף מפתח בעתיד.

# חלק ב' — Resend (שליחת המייל מהטופס)

## מה זה Resend
שירות שמאפשר לאתר לשלוח מיילים (במקרה שלנו — את הפניות מטופס יצירת הקשר).
- **חינם** עד כ-3,000 מיילים בחודש (ועד 100 ביום) — הרבה מעבר לאתר תדמית.
- מאובטח: מפתח ה-API נשמר רק בצד השרת ולעולם לא נחשף בדפדפן.

---

## שלב 1 — הרשמה
1. נכנסים ל-**resend.com** → **Sign Up**.
2. נרשמים עם Google או מייל.
   ⚠️ **חשוב:** הירשם עם **המייל שאליו אתה רוצה שהפניות יגיעו** (למשל המייל של הלקוח).
   בשלב ההתחלתי (לפני אימות דומיין) אפשר לשלוח **רק** למייל הזה.
3. מאמתים את כתובת המייל (לוחצים על הקישור שמגיע לתיבה).

---

## שלב 2 — להוציא מפתח API
1. בתפריט Resend → **API Keys** → **Create API Key**.
2. שם: למשל `tzahi-site`. הרשאה: **Sending access**. יוצרים.
3. **מעתיקים מיד** את המפתח (`re_...`) — הוא מוצג **רק פעם אחת**. אם איבדת — יוצרים חדש.

---

## שלב 3 — לחבר את המפתח ל-Vercel
חוזרים ל-Vercel → הפרויקט → **Settings → Environment Variables**:
1. מוסיפים `RESEND_API_KEY` = המפתח שהעתקת.
2. (אופציונלי) `RESEND_TO` = המייל לקבלת הפניות.
3. שומרים, ואז **Deployments → … → Redeploy** כדי שההגדרה תיכנס לתוקף.

---

## שלב 4 — בדיקה
1. נכנסים לאתר, ממלאים את הטופס עם פרטים אמיתיים, שולחים.
2. בודקים שהמייל הגיע לתיבה — **כולל תיקיית ספאם** בפעם הראשונה.
3. ב-Resend → **Logs** רואים כל מייל שנשלח והאם הצליח.

---

## שלב 5 (מומלץ, לא חובה) — מייל "משם מקצועי" + אימות דומיין 🔧
ברירת המחדל שולחת מ-`onboarding@resend.dev`. כדי שהמייל יישלח מכתובת מותגית
(למשל `office@tzahi-tzedaka.co.il`) ולא ייפול לספאם:

1. ב-Resend → **Domains → Add Domain** → מזינים את הדומיין של הלקוח.
2. Resend נותן מספר רשומות **DNS** (SPF, DKIM, ולפעמים DMARC).
   מוסיפים אותן אצל ספק הדומיין (או ב-Vercel אם הדומיין מנוהל שם).
3. ממתינים לאימות (כמה דקות עד שעות) → סטטוס הדומיין הופך ל-**Verified**.
4. ב-Vercel מעדכנים את `RESEND_FROM` לכתובת בדומיין, ועושים Redeploy.

**יתרון:** המייל נראה מקצועי, מגיע מהעסק, ויש פחות סיכוי שייחסם.

# נספח — פתרון תקלות נפוצות

| תופעה | סיבה סבירה | פתרון |
|---|---|---|
| הטופס מציג "לא הצלחנו לשלוח" ומפנה לוואטסאפ | אין `RESEND_API_KEY` או שגוי | להוסיף/לתקן את המשתנה ב-Vercel ולעשות Redeploy |
| המייל לא מגיע אבל אין שגיאה | המייל בספאם / `RESEND_TO` לא המייל של חשבון Resend | לבדוק ספאם; לפני אימות דומיין חייבים לשלוח למייל של בעל החשבון |
| שינוי במשתנה לא השפיע | משתני סביבה נטענים רק בבנייה | תמיד לעשות **Redeploy** אחרי שינוי משתנה |
| הדומיין לא עובד אחרי חיבור | ה-DNS עוד לא התעדכן | להמתין (עד כמה שעות) ולבדוק שוב; לוודא שהרשומות הוזנו נכון |
| https לא עובד / אזהרת אבטחה | תעודת SSL עוד בהנפקה | Vercel מנפיק אוטומטית; להמתין כמה דקות אחרי חיבור הדומיין |
